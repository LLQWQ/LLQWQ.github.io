无法通过 params 给映射对象中的 Map 类型传值

  • 版本：

    • spring-boot：2.2.13RELEASE

  • 问题描述：

    无法直接在路径地址后面拼接参数给映射对象中map属性传值

    例：http://localhost:8080/list?id=1&role.id=1&map.a=2

    在这个例子中Entity类中的map无法被正确的映射值，但是 role.id 却可以映射到值

    猜测是 路径中的 map.a 无法被映射到正确的类型所以boot就没有对map进行映射，map.a=2 中的 2 可能是 string 也可能是 int 类型

    entity：

    controller：

  • 解决方案

    • 在 controller 中的参数列表中加入 @RequestBody 注解

    • @GetMapping("/list")
@ResponseBody
public String list(@RequestBody Entity entity) {
    return "success";
}

    • 然后前台请求时不要在 params 中传递参数，在 body 中传递参数即可

      • 这样就可以拿到 map.a 的属性值了
