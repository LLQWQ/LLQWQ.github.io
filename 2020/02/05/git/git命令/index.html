git命令行操作

  1. git init

    • 效果：使用该命令会将此文件夹加入git进行管理，随后会在当前目录下生成一个.git隐藏文件夹，这个文件夹就是版本库，而当前目录及其子目录就是工作区。
    • 注意：.git文件会记录你在这个工作区中的操作，切勿删除和修改

  2. git add <file>

    • 作用：将文件从工作区提交到暂存区（就是把文件交给git管）
    • 注意：但此时文件并没有真正进入到版本库当中，文件目前只处于一个中间状态，如果文件没有处于暂存区的话git是不会去记录你的操作的，如果数据被删除了是无法通过git恢复的。

  3. git commit <file>

    • 作用：将暂存区文件提交到版本库中
    • 参数：-m "<message>" 可以不用进入vim编辑器中输入提交信息
    • 注意：这个命令将处于中间状态的文件（暂存区的文件）提交到版本库中,这时才算真正完成了一次提交过程。第一次提交为root（根）提交

    一般情况下这三个命令的使用顺序应该是 init -> add -> commit
    如果此时工作区中的文件发生了改动，并且希望将修改后的文件提交至版本库

    1. 必须先使用git add <file>命令将修改后的文件存入暂存区
    2. 再使用git commit <file>命令将文件从暂存区提交至版本库才可以，这和svn有很大的区别，SVN在添加新文之后每次修改的文件直接commit就可以了。
    3. 如果该文件已经处于追踪状态的话则可以直接使用git commit <file>提交到本地仓库，但是这样就不能撤销了

  4. git status

    • 作用：查看工作区中文件的状态

  5. git rm --cached <file>

    • 作用：将文件从暂存区移除
    • 注意：这条命令并不会删除文件夹中的文件，只是单单的把文件从暂存区中移除（简单的说就是：不把这个文件给git管了）
      ‘rm’ 是(remove)删除的意思，’cached’: 缓存

  6. git reset HEAD <file>

    • 作用：将文件从暂存区移除

版本的前进和后退

查看提交的记录

  • git log

    • 作用：查看提交的记录(最完整的)
    • 参数：
      1. --pretty
        • 值：online
        • 解释：每条log显示一行（简洁一些）
      2. --online
        • 解释：每条log显示一行，但是hash值只显示前一部分，并且只显示后面的版本不显示前面的版本
    • ps：多屏显示的控制方式（空格：向下翻页，b：向上翻页，q：退出）

  • git reflog

      + 作用：和上面的差不多但是会多一列参数'HEAD@{<移动到当前版本需要多少步>}'
  + ps：所有记录都可以显示

修改版本

  • 方式
    • 基于索引值 [推荐]
      • git reset --hard <想去位置的索引值>
      • ps：前进和后退都可以用
    • 使用^符号
      • git reset --hard HEAD^
      • ps：只能后退，如果想后退2个版本就加两个^，以此类推
    • 使用~符号
      • git reset --hard HEAD~1
      • ps：只能后退，要退几步~后就写几

reset命令三个参数的对比

  • --soft
    • 解释：不会管暂存区和工作区，只用于在本地库移动版本指针（也就是指移动版本但）
  • --mixed
    • 解释：在本地库移动版本指针和重置暂存区
  • --hard
    • 解释：在本地库移动版本指针和重置暂存区&工作区

恢复删除的文件

ps：适用于本地仓库有记录的情况
  • 当文件被删除但本地仓库里有该文件的记录则可以使用git reset --hard <文件未被删除时的版本hash>来恢复文件

比较差异文件

  • git diff <file>
    • 解释：不加参数时表示为工作区和缓存区进行比较
    • 参数：
      • HEAD
        • 解释：如果使用版本指针指定某一个版本的话则表示缓存区和指定版本进行比较
    • ps：不带文件名表示比较当前目录的所有文件

分支

创建分支

  • git branch <分支名称>

    查看分支

  • git branch -v

    切换分支

  • git checkout <分支名称>

    合并分支

  • git merge <合并过来的分支名称>
  • ps：假设有A,B两个分支A要合并到B上去，那么我们当前必须要处于B分支，也就是被合并的分支
  • 步骤
    1. 切换到被合并的分支上（执行git checkout <被合并的分支名>）
    2. 执行checkout命令

解决冲突

  1. 解决完文件中的冲突（打开文件，将文件改成想要的样子），保存
  2. 提交到暂存区，使用：git add <file>
  3. 提交到本地仓库，使用：git commit -m "<日志>"
    注意：这里一定不能带文件名

设置签名

  • 介绍

    • 项目/仓库级别：仅在当前本地库的范围内有效
    • 系统用户级别：当前登录的用户范围（可以理解为当前系统用户的全局范围）
    • 优先级：如果两个级别都有设置则就近原则以项目级别的签名为准（项目级别 > 系统用户级别）
    • 注意：如果两者都没有会报错，必须拥有其一

  • 形式

    • 用户名：123qwq
    • email地址：sdjfosidfn@qq.com

  • 作用

    • 区分不同的开发人员的身份

  • 注意

    • 这里设置的签名和登录远程仓库（代码托管中心）的账号和密码是没有半毛钱关系的

  • 命令

    • 增加项目级别：

      git config user.name tom_pro --add
      git config user.email good1245_pro.@qq.com --add

    • 增加全局：

      git config user.name tom_pro --add --global
      git config user.email good1245_pro.@qq.com --add --global

    • 删除：

      git config user.name --unset

    • 改：

      git config user.name qwq123

    • 查看签名：

      cat .git/config
      git config --list

    • 查看指定字段：

      git config user.name

    • 注意：这里的意思是打开.get文件夹下的config文件，如果不想使用命令查看签名也可以用记事本打开config文件，修改也一样.
      设置或查看全局的签名在后面加--global 即可，但是如果使用cat命令就得到当前系统用户的文件夹下查看使用cd ~即可到达，然后再使用cat .\.gitconfig即可查看
      增加和修改命令其实差不多

与第三方远程库进行交互

  • 保存项目地址
    • 查看地址别名：git remote -v
    • 新建一个地址别名：git remote add <别名> <仓库地址>
  • 从远程仓库下载项目
    git clone <仓库地址>
    ps：clone有三个效果
    1. 不需要进行git init仓库已经初始化好了
    2. 不需要保存项目地址，会自动添加（origin）
    3. 完整的把远程库下载到本地
  • 将本地仓库提交到远程库
    • git push <别名> <推送的分支>
    • ps：如果当前提交的这个项目自己不在这个项目的团队中的话会提交失败
  • 拉取远程仓库中的更新
    • git pull <别名> <抓取的分支>
    1. git fetch <别名> <抓取的分支>
      • 这一步只是将远程仓库中的内容抓取下来，但是还没有对我们本地工作区进行修改，如果需要要切换到对应的版本，使用 git checkout <别名>/<分支名>即可
    2. 切换到本地工作区中被合并的分支上 git checkout <被合并的分支>
    3. 将远程仓库的分支合并到需要被合并的分支上 git merge <别名>/<被合并的分支>
      ps：pull = fetch+merge，如果更新的远程数据比较简单、不太会产生冲突的话使用pull, 但是更新的数据比较复杂时推荐使用分步操作。
      如果两个人修改了同一个地方，第一个提交到远程仓库的人可以提交成功，但是第二个就不行。这种情况第二个人需要
    4. 将本地工作区中的更改提交到本地仓库
    5. 将代码拉取来下
    6. 解决冲突
    7. 提交

跨团队协作

有A、B两个人，A找B开发项目但是B不能跳槽到A的团队去

  1. B把A的项目fork一份
  2. B把fork下来的项目clone到本地
  3. B完成A给的任务，fork下来的远程库上
  4. B发起pull request
  5. A审核，审核ok后合并分支

设置ssh登录

  1. 进入用户的根目录 cd ~
  2. 删除以前的ssh rm -r .ssh/ 如果没有就不用删
  3. 生成ssh ssh-keygen -t rsa -C <第三方仓库的账号（邮箱）> C是大写的！！
  4. 进入到.ssh目录中 cd ./.ssh/
  5. 打开目录中的id_rsa.pub文件 cat id_rsa.pub 并复制里面的内容
  6. 进入到第三方仓库中设置ssh公钥
  7. 新建一个远程ssh连接的别名 git remote add <别名> <ssh地址>
  8. 使用ssh登录就要将推送的别名更改为刚刚设置的ssh的别名

eclipse Git插件

  • 将工程初始化为一个git工程（右键->theam->share project->git）

git工作流

  • A为分发任务的，B为程序员
  1. 创建主分支
  2. A分发一个需求到B，B创建一个分支（new）用来完成这个功能
  3. B完成之后提交到远程库
  4. A拉取远程库中的new分支，检查有没有bug，这个功能通不通过，如果不通过就叫A改，如果通过就合并到主分支
