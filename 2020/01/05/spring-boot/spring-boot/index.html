配置文件位置

配置文件默认加载位置

spring-boot默认从以下位置加载配置文件

  1. file:./config/
  2. file:./config/*/
  3. file:./
  4. classpath:/config/
  5. classpath:/

优先级从上到下

spring-boot会从以上路径从上到下依次加载；如果配置文件中有相同的属性则会以高优先级的为准

修改默认配置文件名称

spring-boot默认会去查找名称为application的配置文件

如果不喜欢默认的application名称，可以使用spring.config.name来更改默认名称

指定配置文件位置

spring.config.location

spring-boot中可以使用spring.config.location来指定配置文件的位置，默认位置配置将被替代

例如，如果 spring.config.location 配置为 classpath:/custom-config/,file:./custom-config/，搜索顺序将变为以下：

  1. file:./custom-config/
  2. classpath:custom-config/

==该属性在spring项目的配置文件中指定是无效的，必须使用命令行参数指定==

命令行指定的配置文件则指定的那个配置文件为最高优先级

使用方法：

将项目打包后使用如下命令

java -jar xxx.jar --spring.config.location=d:a/qwq.yml

如果指定的是一个目录而不是文件则会查找该目录下的application.yml/.properties

⚠️注意：如果指定的是一个目录必须在末尾写上‘/’；

例：

​ spring.config.location=d:config/ (√)

​ spring.config.location=d:config (×)

spring.config.additional-location

或者，当使用 spring.config.additional-location 配置自定义配置位置时，除了使用默认位置外，还会使用它们。这些其他（additional）位置将在默认位置之前搜索。例如，如果将其他位置配置为 classpath:/custom-config/,file:./custom-config/，则搜索顺序将变为以下内容：

  1. file:./custom-config/
  2. classpath:custom-config/
  3. file:./config/
  4. file:./
  5. classpath:/config/
  6. classpath:/

该搜索顺序允许您在一个配置文件中指定默认值，然后有选择地覆盖另一个配置文件中的值。您可以在 application.properties（或您使用 spring.config.name 指定的其他文件）中的某个默认位置为应用程序提供默认值。之后，在运行时，这些默认值将被自定义位置中的某个文件所覆盖。

Thymeleaf中使用th:if 或者 th:unless ，两者的意思恰好相反。

如果表达式的值为true，则标签会渲染到页面，否则不进行渲染。

以下情况被认定为true：

  • 表达式值为true
  • 表达式值为非0数值
  • 表达式值为非0字符
  • 表达式值为字符串，但不是"false","no","off"
  • 表达式不是布尔、字符串、数字、字符中的任何一种

其它情况包括null都被认定为false

  3. 后置处理器，执行postProcessBeforeInitialization（）方法，调用定制器的定制方法；

内置servlet容器启动原理

  1. 项目启动，springboot调用run方法

  2. 调用run()过程中调用refreshContext(context);创建并刷新ioc容器

    1. 调用createApplicationContext();
    2. 判断当前项目类型，根据项目类型创建对应的ioc容器
    1. 调用((AbstractApplicationContext) applicationContext).refresh();方法刷新ioc容器
    2. 因为创建的是嵌入式servlet ioc容器所以调用的就是servlet ioc容器的refresh()方法，servlet ioc容器重写了refresh()
    3. 调用servlet ioc容器的this.onRefresh();方法
    4. 调用createWebServer();来创建并启动嵌入式servlet容器
